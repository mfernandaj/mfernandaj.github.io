Tutorial Angular para principiantes

Por Fernanda Jiménez

Aprende Angular desde Cero

Bienvenido a este tutorial interactivo de Angular para principiantes.

Descubre cómo crear aplicaciones dinámicas y estructuradas con las mejores prácticas.


¿Qué es Angular y para qué sirve?

Angular es un poderoso framework de código abierto desarrollado y mantenido por Google, diseñado para crear aplicaciones web dinámicas y modernas. Facilita la creación de aplicaciones de una sola página (SPA), donde el procesamiento ocurre principalmente en el navegador, mejorando la experiencia del usuario al minimizar las recargas de página.

Aplicaciones destacadas de Angular

Angular es ideal para desarrollar aplicaciones web interactivas y escalables. Sus aplicaciones más comunes incluyen:

  • Plataformas empresariales de gestión interna.
  • Sitios de comercio electrónico con experiencia de usuario fluida.
  • Aplicaciones móviles híbridas utilizando herramientas como Ionic.
  • Dashboards interactivos para análisis de datos en tiempo real.

¿Qué tecnologías utiliza Angular?

Angular se basa en TypeScript, un superconjunto de JavaScript que incorpora tipado estático y herramientas avanzadas. Esto permite a los desarrolladores escribir código más organizado, seguro y mantenible. Además, utiliza HTML para la estructura de las vistas y CSS para el diseño visual, combinados con herramientas modernas como RxJS para la programación reactiva.

Requisitos previos para trabajar con Angular

Antes de comenzar con Angular, se recomienda tener conocimientos básicos en:

  • HTML y CSS, para el diseño y la estructura de las vistas.
  • JavaScript, el lenguaje base del desarrollo web moderno.
  • TypeScript (opcional, pero altamente recomendable).
  • Node.js y npm, necesarios para la instalación y gestión de paquetes.
  • Desarrollo de aplicaciones web, con énfasis en las SPA.

Instalación

Aprende cómo instalar y configurar Angular paso a paso. En esta sección te guiaré en el proceso de instalación de Angular para Windows. Siguiendo los siguientes pasos:

Paso 1. Requisitos previos: Instalar Node.js y npm

Asegúrate de contar con lo siguiente antes de instalar Angular:

Node.js proporciona un entorno para ejecutar JavaScript fuera del navegador, mientras que npm (Node Package Manager) permite gestionar las dependencias del proyecto, como Angular CLI.

Es fundamental asegurarte de que tienes instaladas versiones compatibles:

  • Node.js: versión 14.20.0 o superior.
  • npm: versión 6.14.0 o superior.
  • Node.js y npm. Descárgalos desde la página oficial de Node.js.
  • Verifica su instalación con los comandos:
    • node -v y npm -v

    Paso 2. Instalar Angular CLI

    La CLI (Command Line Interface) de Angular permite crear y gestionar proyectos fácilmente. Para instalarla, ejecuta el siguiente comando:

    npm install -g @angular/cli

    Este comando instala la CLI de forma global en tu sistema. Tardará unos minutos.

    Paso 3. Comprobar la instación ejecutando comandos CLI de Angular

    Luego comprueba ejecutando el siguiente comando que se ha instalado correctamente: 

    ng --version

    Deberías ver una salida que incluye la versión de Angular CLI, Node.js y otros paquetes relacionados.

    Paso 4: Crear un nuevo proyecto Angular

    Una vez instalada la CLI, crea un nuevo proyecto con:

    ng new nombre-del-proyecto

    Durante el proceso, se te preguntará:

    ¿Desea habilitar la representación del lado del servidor (ssr) y la generación de sitios estáticos (ssg/prerendering)? Responde Yes si deseas configurar rutas desde el inicio.
    ¿Qué formato de hoja de estilo te gustaría usar? Elige entre CSS, SCSS, SASS o LESS.

    Versiones necesarias

    1. Angular requiere una versión específica de Node.js para funcionar correctamente. Actualmente, se recomienda:

    • Node.js >= 16.14.0.

    Es buena práctica instalar una versión LTS (Long Term Support), ya que es más estable.

    2. NPM (Node Package Manager) incluido con Node.js. Asegúrate de tener una versión actualizada.

    • NPM >= 6.11.0.

    3. Angular CLI

    • Instala la última versión estable de Angular CLI.

    4. TypeScript, Angular requiere una versión específica de TypeScript. La versión mínima recomendada dependerá de la versión de Angular que estés instalando. Por ejemplo

    • Angular 16 requiere TypeScript 5.0+.

    • Angular 15 requiere TypeScript 4.8+.

    5. Navegador moderno

    • Angular funciona en navegadores modernos como Chrome, Firefox o Edge.

    Rutas para tener en cuenta

    • Los proyectos se crean en cualquier carpeta, pero evita espacios y caracteres especiales en las rutas.

    Variables de entorno

    • No es necesario configurar variables especiales para un proyecto base.

    Se necesita de base de datos?

    • No es obligatorio para proyectos simples. Si trabajas con APIs, puedes integrar bases de datos como MySQL, MongoDB, etc.

    Lo esencial es tener configurados los requisitos previos (Node.js y npm) y la instalación de Angular CLI

    Primeros Pasos

    Vamos a crear nuestro primer componente en Angular que mostrará el clásico "Hola Mundo". Este es un ejemplo sencillo pero fundamental para entender cómo funciona Angular.

    1. Asegúrate de tener instalado Angular CLI. Puedes verificarlo usando: 
      ng --version
    2. Crea un nuevo proyecto Angular con el siguiente comando: 
      ng new hola-mundo
    3. Una vez creado el proyecto, navega a la carpeta y ejecuta el servidor local: 
      cd hola-mundo
      ng serve
      Abre tu navegador en http://localhost:4200
      .
    4. Edita el archivo src/app/app.component.ts, el template y reemplaza o comenta su contenido con: 
      <h1>Hola Mundo con Angular!</h1>
    5. Guarda los cambios y actualiza tu navegador. Deberías ver el mensaje "¡Hola Mundo con Angular!" en pantalla.

    Utilización del Framework

    ¿Cómo empezar a crear una aplicación base?

    Este tutorial que estás leyendo es, de hecho, una aplicación Angular. Para crear una aplicación base similar, sigue los siguientes pasos:

    1. Asegúrate de tener instalada la CLI de Angular. Si aún no lo has hecho, consulta la sección de instalación de este tutorial para más detalles.
    2. Utiliza el comando siguiente para generar un nuevo proyecto: 
      ng new tutorial-angular

      Este comando creará la estructura inicial del proyecto en una carpeta llamada tutorial-angular.

    Ahora tienes una aplicación base lista para personalizar. Este tutorial se desarrolló utilizando las mismas herramientas y pasos descritos aquí, lo que demuestra lo rápido y eficiente que es Angular para crear aplicaciones estructuradas.

    ¿Cómo visualizar los resultados de la ejecución?

    Para ver tu aplicación en funcionamiento, sigue estos pasos:

    1. Desde la carpeta del proyecto, ejecuta el siguiente comando: 
      ng serve

      Esto iniciará el servidor de desarrollo local.

    2. Abre tu navegador y accede al enlace proporcionado en la terminal, normalmente: 
      http://localhost:4200
    3. Verás la página predeterminada de Angular, que ahora está lista para ser personalizada. Si deseas replicar el diseño de este tutorial, sigue las secciones avanzadas de este documento para modificar componentes y estilos.

    Recuerda que Angular CLI actualizará automáticamente la página en el navegador cada vez que realices cambios en el código.

    ¿Cómo crear componentes?

    En Angular, los componentes son las unidades principales de la interfaz de usuario. Para crear un componente como los utilizados en este tutorial (por ejemplo, los del menú de navegación), sigue estos pasos:

    1. Genera un nuevo componente utilizando Angular CLI con el siguiente comando: 
      ng generate component nombre-del-componente

      Esto creará automáticamente la carpeta del componente con sus archivos: HTML, CSS, TypeScript

    2. Ejemplo práctico: Los componentes que forman del menú de navegación en este tutorial, cada uno fue generado usando el comando: ng generate component.
    3. Importa y usa los componentes en el archivo de rutas 
      app.routes.ts o app.module.ts si no es standalone:
        
  // Ejemplo en app.module.ts
  import { FrameworkComponent } from './framework/framework.component';
  
  @NgModule({
    declarations: [FrameworkComponent],
    imports: [],
    bootstrap: [AppComponent]
  })
  export class AppModule { }
    4. Referencia el componente en la plantilla principal mediante su selector: 
      <app-framework></app-framework>

    Siguiendo estos pasos, puedes crear componentes modulares reutilizables que estructuran y organizan tu aplicación Angular.

    Explicación del funcionamiento del framework

    ¿Actúa sobre páginas, ficheros, secciones…?

    Angular organiza y gestiona las aplicaciones mediante **componentes**, que representan partes de la interfaz de usuario. Cada componente es una combinación de: 

    • HTML: Define la estructura visual (elementos, contenedores, botones, etc).
    • CSS o SCSS: Añade estilos específicos que afectan solo a ese componente.
    • TypeScript: Contiene la lógica del componente (eventos, propiedades, funciones).

    Estos componentes se combinan dentro de un árbol jerárquico para construir toda la aplicación, desde pequeñas secciones hasta páginas completas.

    ¿Cómo se editan/modifican los ficheros que actúan en el framework?

    Los ficheros que conforman Angular pueden editarse fácilmente con un editor como **Visual Studio Code**. Su organización es modular, facilitando las modificaciones: 

    • **HTML**: Se encuentra en la carpeta src/app/ con el sufijo .component.html.
    • **Lógica (TypeScript)**: Localizada en los archivos .component.ts, donde se define la funcionalidad del componente.
    • **Estilos (CSS o SCSS)**: Se editan en los archivos .component.css o .component.scss.

    ¿Cómo se referencian los ficheros creados desde otras aplicaciones (si es el caso)?

    Angular permite reutilizar ficheros y componentes mediante diferentes enfoques, dependiendo de la arquitectura del proyecto. A continuación, se detallan las opciones principales:

    Módulos compartidos

    Para proyectos no standalone, Angular utiliza módulos (`app.module.ts`) para agrupar componentes, servicios y directivas que pueden exportarse e importarse en otros módulos o aplicaciones. 

      
    <!-- Importar módulos en Angular -->
    import { NgModule } from '@angular/core';
    import { CommonModule } from '@angular/common';
    import { MiComponente } from './mi-componente/mi-componente.component';
  
    @NgModule({
      declarations: [MiComponente],
      exports: [MiComponente],
      imports: [CommonModule]
    })
    export class SharedModule {

    Luego, este módulo puede importarse en cualquier otro módulo de la aplicación: 

      
  import { SharedModule } from './shared/shared.module';
  
  @NgModule({
    imports: [SharedModule],
    })
  export class FeatureModule { }

    Proyectos standalone

    En proyectos standalone Angular utiliza (`app.routes.ts`), los componentes y servicios se registran directamente en las rutas o en el componente principal. En este caso, puedes importar un componente desde otro proyecto de esta manera: 

      
  import { MiComponente } from './mi-componente/mi-componente.component';
  
  const routes: Routes = [
  {
      path: 'mi-componente',
      component: MiComponente, // Referencia directa al componente
      }
  ];

    Esto simplifica la gestión de dependencias y reduce la cantidad de archivos adicionales. Es importante tener en cuenta que las aplicaciones standalone están disponibles a partir de Angular 14.

    Bibliotecas

    Angular CLI permite crear bibliotecas reutilizables que pueden compartirse entre proyectos e incluso publicarse en npm. Las bibliotecas son ideales para agrupar funcionalidad común, como componentes UI, servicios o utilidades.

    Puedes generar una biblioteca con el siguiente comando: 

    ng generate library mi-biblioteca

    Una vez creada, la biblioteca puede importarse en otros proyectos Angular mediante npm. Ejemplo de importación en un módulo: 

      
  import { MiBibliotecaModule } from 'mi-biblioteca';
  @NgModule({
    imports: [MiBibliotecaModule],
    })
  export class AppModule { }

    Conclusiones

    A lo largo de este tutorial, hemos explorado los conceptos esenciales de Angular y cómo aplicarlos en un proyecto práctico. Ahora cuentas con las herramientas necesarias para:

    • Iniciar y estructurar proyectos Angular: Has aprendido a crear una aplicación base usando Angular CLI, configurando la estructura inicial del proyecto.

    • Crear y organizar componentes: Vimos cómo generar componentes modulares utilizando el comando ng generate component y cómo estructurarlos dentro de tu aplicación.

    • Utilizar rutas para navegación: Aprendiste a configurar y gestionar la navegación entre secciones usando archivos routes.ts o module.ts, según tu enfoque de proyecto.

    • Reutilizar código con módulos y bibliotecas: Entendiste cómo agrupar y exportar componentes mediante módulos compartidos y cómo crear bibliotecas reutilizables con Angular CLI.

    • Aplicar prácticas recomendadas: La organización modular, separación de lógica (TypeScript), estructura (HTML) y estilos (CSS) te permite mantener aplicaciones escalables y mantenibles.

    Con estos fundamentos, estás preparado para afrontar proyectos más complejos y profundizar en características avanzadas como servicios, directivas, formularios reactivos y consumo de APIs.

    ¡Sigue practicando, explorando documentación oficial y construyendo tus propios proyectos para consolidar tu aprendizaje! Angular ofrece un mundo de posibilidades para crear aplicaciones web modernas y robustas.
