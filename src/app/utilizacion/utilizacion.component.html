¿Cómo empezar a crear una aplicación base?

Este tutorial que estás leyendo es, de hecho, una aplicación Angular. Para crear una aplicación base similar, sigue los siguientes pasos:

  1. Asegúrate de tener instalada la CLI de Angular. Si aún no lo has hecho, consulta la sección de instalación de este tutorial para más detalles.
  2. Utiliza el comando siguiente para generar un nuevo proyecto: 
    ng new tutorial-angular

    Al finalizar la creación del proyecto, te mostrará un mensaje "Packages installed successfully"

    Este comando creará la estructura inicial del proyecto en una carpeta llamada tutorial-angular.

Ahora tienes una aplicación base lista para personalizar. Este tutorial se desarrolló utilizando las mismas herramientas y pasos descritos aquí, lo que demuestra lo rápido y eficiente que es Angular para crear aplicaciones estructuradas.

La estructura básica del proyecto incluye:

  • src/: Contiene el código fuente del proyecto.
  • app/: Carpeta principal donde se encuentran los componentes, módulos y estilos del proyecto.
  • angular.json: Archivo de configuración del proyecto Angular.
  • package.json: Archivo donde se listan las dependencias instaladas y configuraciones del proyecto.

    • ¿Cómo visualizar los resultados de la ejecución?

    Para ver tu aplicación en funcionamiento, sigue estos pasos:

    1. Desde la carpeta del proyecto, ejecuta el siguiente comando: 
      ng serve

      Esto iniciará el servidor de desarrollo local.

    2. Abre tu navegador y accede al enlace proporcionado en la terminal, normalmente: 
      http://localhost:4200
    3. Verás la página predeterminada de Angular, que ahora está lista para ser personalizada.

    Recuerda que Angular CLI actualizará automáticamente la página en el navegador cada vez que realices cambios en el código.

    ¿Cómo crear componentes?

    En Angular, los componentes son las unidades principales de la interfaz de usuario. Para crear un componente como los utilizados en este tutorial (por ejemplo, los del menú de navegación), sigue estos pasos:

    1. Genera un nuevo componente utilizando Angular CLI con el siguiente comando: 
      ng generate component nombre-del-componente

      Esto creará automáticamente la carpeta del componente con sus archivos: HTML, CSS, TypeScript

    2. Ejemplo práctico: Los componentes que forman del menú de navegación en este tutorial, cada uno fue generado usando el comando: ng generate component.

    Organización de recursos multimedia

    Por defecto, Angular no crea la carpeta destinada a almacenar las imágenes y recursos estáticos dentro de la estructura del proyecto. Sin embargo, Angular utiliza esta carpeta si decides crearla manualmente. Al hacerlo, Angular sabe que los archivos dentro de la carpeta `assets` son recursos estáticos que serán copiados al directorio de salida (`dist/`) cuando se ejecute el comando de construcción de la aplicación (`ng build`).

    En este caso, se creó la carpeta `assets/img` dentro de la estructura del proyecto, la cual alberga todas las imágenes necesarias, como las de perfil, logotipos y otros recursos visuales que se utilizan en los componentes del tutorial. Esto asegura una mejor organización y facilita el acceso a los archivos dentro del proyecto.

    Compilación del proyecto y publicación en GitHub Pages

    Para compilar el proyecto Angular y prepararlo para su publicación en **GitHub Pages**, puedes utilizar el siguiente comando:

    ng build --configuration production --output-path docs --base-href /

    A continuación, se explican los parámetros del comando:

  • configuration production:

    Compila la aplicación en modo producción, optimizando los recursos para su despliegue. Esto incluye minificación de archivos, eliminación de código innecesario y optimización de rendimiento.

  • --output-path docs:

    Especifica la carpeta de salida donde se almacenarán los archivos generados tras la compilación. En este caso, se utiliza docs porque GitHub Pages permite servir contenido directamente desde esta carpeta en el repositorio.

  • --base-href /:

    Define el prefijo de las rutas de la aplicación. Al ser publicado en GitHub Pages, este parámetro asegura que las rutas relativas funcionen correctamente.

    Como resultado, Angular genera la carpeta **docs**, que contiene los archivos estáticos necesarios para la publicación (HTML, CSS, JS). Finalmente, esta carpeta se sube al repositorio en **GitHub** para que la aplicación quede disponible en **GitHub Pages**.

    Configuración en GitHub Pages

    Para la publicación del tutorial en GitHub Pages, tras subir los cambios en el repositorio, sigue los siguientes pasos:

  • 1. Ir a la sección Settings (Configuración) de tu repositorio en GitHub.
  • 2. En el apartado Pages, seleccionar Branch: main y la carpeta /docs como fuente.
  • 3. Guardar los cambios.

    De esta manera, tu aplicación Angular estará disponible a través de GitHub Pages.
